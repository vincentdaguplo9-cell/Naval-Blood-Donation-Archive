Technical Process Summary

  • Frontend: JavaFX UI handles input, validation, and navigation.
  • Business Logic: DAO layer executes prepared statements via JDBC.
  • Data Flow: UI event → DAO method → SQL → DB → UI refresh.
  • Security: Hashing + lockout in AdminDAO.
  • UX Controls: Disabled buttons until valid input, inline errors.

Technical Q&A (App‑Focused)

  • How do you prevent invalid input? Required fields, inline errors, and disabled action buttons.
  • How do you avoid SQL injection? Prepared statements in DAO layer.
  • How is the donation flow handled? One UI action creates unit + transaction, then updates eligibility.
  • Why pagination? Keeps table rendering responsive for large datasets.
