Speaker Scripts

Daguplo

“This defense focuses on the application layer. NBDA is built with JavaFX for UI and JDBC for data access. The architecture follows a clear separation: UI → DAO → Database.”

“Each UI action triggers DAO operations using prepared statements, ensuring consistent validation and safe database interaction.”

Lamoste

“Navigation flow: Login → Dashboard → modules (Donors, Donation, Inventory, Reports). The dashboard summarizes totals and recent transactions for operational awareness.”

“We designed the UI to be keyboard-friendly with default buttons and logical focus order.”

Ocampo

“Donor Management: CRUD with search and pagination. Donor edits update the master record without losing transaction history.”

“Record Donation: creates a blood_unit and donation_transaction in a single flow, then updates donor eligibility.”

Fuentes

“Reports aggregate data across tables. We export CSV for analysis and use printing for PDF generation.”

“Pagination keeps the UI responsive even with larger datasets.”

Cator

“Security includes hashed passwords and lockout after repeated failed attempts. Input validation prevents incomplete or invalid submissions.”

“Buttons are disabled until required fields are valid, improving data quality and user experience.”

Key App Features

  • Secure login with password hashing and lockout.
  • Donor CRUD with search and pagination.
  • Donation recording with auto inventory updates.
  • Reports with CSV/PDF export.
